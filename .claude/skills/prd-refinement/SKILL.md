# PRD Refinement Skill (요구사항 명확화 및 최소 업데이트)

이 스킬은 개발 착수 전, PRD 문서의 누락되거나 모호한 부분을 식별하여 구체화하고 문서의 일관성을 유지하는 역할을 수행합니다. **PRD의 메인 작성자는 다른 AI 에이전트(`Android-PRD-Builder`)이므로, 문서 전체를 덮어쓰거나 구조를 변경해서는 안 됩니다.**

## 1. 🔍 식별 (Identify)
PRD를 읽고 다음과 같은 '개발 시 병목이 될 수 있는 모호한 항목'을 찾아냅니다.
- **Figma 연동 누락**: 기능에 대응하는 Figma Node ID 또는 시안 링크가 없는 경우.
- **상태/이벤트 모호성**: `UiState`의 기본값, 예외 발생 시 상태 전이(State Mutation) 로직이 불명확한 경우.
- **리소스(Resource) 미정의**: 아이콘(`R.drawable.xxx`), 텍스트(`R.string.xxx`), 색상(`Color`) 등 구체적인 리소스 식별자가 지정되지 않은 경우.
- **테스트 기준 부재**: 단위 테스트의 Given-When-Then 구조나 수동 테스트의 트리거/피드백 조건이 추상적인 경우.

## 2. 🙋‍♂️ 명확화 질문 (Ask User)
식별된 모호한 점들을 모아 사용자에게 구체적인 객관식 또는 주관식 질문을 던집니다. `AskUserQuestion` 도구를 사용하거나, 서브에이전트 컨텍스트에서는 아래 `<clarification_required>` 형식으로 질문하고 멈춥니다.

```
<clarification_required>
[근거] 무엇이/왜 모호한지 (PRD 항목·코드 파일:라인 명시)
[질문] 확답이 필요한 1~2개의 구체적 질문
[선택지] (가능하면) 고를 수 있는 후보안 + 각 트레이드오프
</clarification_required>
```

- **예시**: "F-01 기능의 뒤로가기 버튼 동작 시, 변경되어야 할 UiState 필드가 명시되지 않았습니다. A) 이전 상태 유지, B) 초기화 중 어떤 것을 원하시나요?"
- **예시**: "해당 다이얼로그에 사용될 아이콘 리소스 이름이 없습니다. `ic_warning`을 사용할까요, 아니면 새로 추가할까요?"

## 3. ✍️ 최소한의 문서 업데이트 (Surgical Update)
사용자의 답변을 받으면, **반드시 `Edit` 도구만을 사용**하여 PRD 문서에서 딱 필요한 줄(Line)이나 표의 셀(Cell)만 부분적으로 수정합니다.
- **절대 금지**: `Write` 도구를 사용하여 PRD 문서 전체를 다시 작성하거나 덮어쓰지 마세요.
- **원칙**: 원본 작성자(다른 AI)의 어조, 포맷, Markdown 테이블 구조를 100% 보존하세요.

## 4. 🚀 개발 진행 핸드오버
업데이트가 완료되면 "PRD 명확화 및 문서 동기화를 완료했습니다."라고 선언한 뒤, 코드 설계 및 구현(Orchestrate & Implement) 단계로 넘어갑니다.
