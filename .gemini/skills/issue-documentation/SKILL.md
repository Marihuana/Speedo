# Issue Documentation Skill

이 스킬은 사용자가 보고한 문제를 정밀하게 파악하고, 이를 `PRD.md` 또는 관련 문서에 '이슈 사항'으로 정리하여 개발 에이전트가 정확한 수정을 진행할 수 있도록 돕습니다.

## 1. 🔍 이슈 파악 및 질문 (Issue Clarification)
- **현상 수집:** 사용자가 보고한 문제 상황(Expected vs Actual)을 명확히 파악합니다.
- **정보 부족 판단:** 문제 재현이나 원인 분석을 위해 추가 정보(재현 경로, 특정 기기 환경, 로그 등)가 필요하다고 판단되면 사용자에게 질문하여 명확히 합니다.
- **중립성 유지:** 성급하게 원인을 단정하거나 특정 해결책(How)을 설계하지 않습니다.

## 2. 📝 이슈 정리 (Technical Issue Documentation)
- **현상 기술 (Symptom):** 사용자가 겪고 있는 문제를 관찰 가능한 현상 중심으로 기술합니다.
- **기술적 컨텍스트 (Context):** 해당 문제와 관련된 기존 코드, 데이터 구조, 센서 특성 등을 나열합니다. 
  - 예: "TelemetryRepositoryImpl.kt의 집계 로직이 위치 수신 주기와 연동되어 있음"
- **불일치 지점 명시 (Discrepancy):** 실시간 대시보드와 저장된 로그 간의 차이 등 불일치가 발생하는 지점을 명확히 짚어줍니다.

## 3. ✍️ PRD 업데이트 (PRD Integration)
- `PRD.md`의 `§4. Exception Handling & Edge Cases` 또는 관련 섹션에 '이슈 사항' 항목을 추가하거나 업데이트합니다.
- 기존 기능을 수정하는 것이 아니라, 해결해야 할 '문제'를 요구사항의 일부로 등록합니다.
- 근거 명세 시 관련 파일 경로와 심볼명을 포함하여 개발 에이전트의 분석을 돕습니다.

## 4. 🚀 출력 및 승인 (Output & Approval)
- **사전 보고:** `PRD.md` 파일에 이슈 사항을 등록하기 전, 정리된 이슈의 핵심(현상, 기술적 컨텍스트, 관련 파일 등)을 사용자에게 먼저 보고합니다.
- **승인 후 반영:** 사용자의 명시적인 승인이 있은 후에만 `write_file` 또는 `replace` 도구를 사용하여 파일에 반영합니다.
- 최종 결과물은 `PRD.md` 파일에 반영하며, 해결책보다는 **"무엇이 문제인가"**를 정의하는 데 집중합니다.
